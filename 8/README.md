# Лабораторная работа №8: Аннотации и Stream API

## Задание: DataProcessor.java, DataManager.java, Main.java, FilterProcessor.java, TransformProcessor.java, AggregateProcessor.java

Приложение для многопоточной обработки данных с использованием аннотаций, рефлексии и Stream API.

### Структура проекта

- `DataProcessor.java` — определение пользовательской аннотации `@DataProcessor`, которая используется для пометки методов обработки данных. Аннотация имеет политику удержания `RUNTIME`, что позволяет обращаться к ней через рефлексию во время выполнения программы.

- `DataManager.java` — основной класс управления данными. Содержит:
  - `registerDataProcessor(Object processor)` — регистрирует объект-обработчик, сканирует его методы с помощью рефлексии и сохраняет те, которые помечены аннотацией `@DataProcessor`.
  - `loadData(String source)` — загружает данные из указанного источника (например, из файла или списка).
  - `processData()` — запускает многопоточную обработку данных, применяя зарегистрированные методы-обработчики к каждому элементу данных с использованием Stream API и `parallelStream()` для параллельной обработки.
  - `saveData(String destination)` — сохраняет обработанные данные в указанный источник.

- `FilterProcessor.java` — класс-обработчик для фильтрации данных. Содержит метод, помеченный аннотацией `@DataProcessor`, который оставляет только те элементы, которые удовлетворяют заданному условию (например, числа больше определённого порога).

- `TransformProcessor.java` — класс-обработчик для трансформации данных. Содержит метод, помеченный аннотацией `@DataProcessor`, который преобразует каждый элемент данных (например, умножает числа на 2 или переводит строки в верхний регистр).

- `AggregateProcessor.java` — класс-обработчик для агрегации данных. Содержит метод, помеченный аннотацией `@DataProcessor`, который выполняет свёртку данных (например, вычисляет сумму всех элементов или объединяет строки).

- `Main.java` — демонстрационный класс, который:
  - Создаёт экземпляры всех обработчиков (`FilterProcessor`, `TransformProcessor`, `AggregateProcessor`).
  - Регистрирует их в `DataManager`.
  - Загружает исходные данные.
  - Запускает многопоточную обработку.
  - Сохраняет и выводит результат.

### Принципы работы

1. **Аннотации и рефлексия:** Методы обработки помечаются аннотацией `@DataProcessor`, а `DataManager` динамически находит их через рефлексию во время выполнения.

2. **Многопоточность:** Для параллельной обработки данных используется `ExecutorService` и `parallelStream()`, что позволяет эффективно загружать все ядра процессора.

3. **Stream API:** Фильтрация, трансформация и агрегация данных выполняются с использованием функциональных операций Stream API (`filter()`, `map()`, `collect()`, `reduce()`).

4. **Гибкость:** Система легко расширяется — достаточно создать новый класс с методами, помеченными `@DataProcessor`, и зарегистрировать его в `DataManager`.
